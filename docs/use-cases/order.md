# Use Case: Замовлення користувача

## Опис

Даний Use Case описує процес оформлення замовлення користувачем.

## Ціль

Отримати можливіть замовляти товари з метою подальшого отримання їх.

## Дійові особи

* **Гість** - неавторизований відвідувач сайту.
* **Користувач** - зареєстрований та авторизований користувач.

## Передумови

* Користувач знаходиться на сторінці каталогу або всередині самого кошика.

### Основний сценарій: Успішне оформлення

1. Користувач натискає посилання `Proceed to Checkout`.
2. Система відображає сторінку з формою контактних даних для оформлення замовлення.
3. Користувач вводить свої контактні дані, натискає `Proceed to Shipping`.
4. Система валідує дані, створює замовлення, переносить в нього всі товари з поточного кошика користувача, відображає сторінку підтвердження замовлення, якщо користувач авторизований, то записує ще його унікальний ідентифікатор в базу даних.
5. Користувач натискає `Complete Order`.
6. Система оновлює статус замовлення користувача в базі даних, повністю очищає кошик та перенаправляє його на сторінку успішного оформлення замовлення де відображається номер замовлення та статус.

### Альтернативні сценарії:

**A1: Кошик пустий**

- 1.a. Система виявляє, що кошик не містить жодного товару.
- 2.б. Система скасовує перехід та робить редірект назад на головну сторінку.
