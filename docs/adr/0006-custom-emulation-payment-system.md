# ADR 0006: Кастомна емуляція платіжної системи

## Контекст і постановка проблеми

Потрібно реалізувати обробку оплати замовлень.

## Драйвери прийняття рішень

* Швидкість розробки: запуск MVP проєкту.

## Розглянуті варіанти

1. Інтеграція реальної платіжної системи.
2. Власна емуляція оплати.

## Результат рішення

Ми вирішили обрати **Варіант 2**. Емуляція дозволяє протестувати потрібну логіку без витрат часу на інтеграцію реальної платіжної системи.

### Наслідки

#### Плюси

- Можливість розробляти та тестувати бізнес-логіку оплати незалежно від зовнішнього сервісу.

#### Мінуси

- Необхідність рефакторингу для заміни емулятора на реальну платіжну систему.
