# ADR 0003: Пошукова система за допомогою Laravel Scout (Database Engine)

> **Увага:** Це рішення було скасовано та замінено на [ADR 0004](0004-custom-search-and-filter-system.md).

## Контекст і постановка проблеми

Потрібно реалізувати функціонал пошуку товарів у каталозі з можливістю подальшої фільтрації.

## Драйвери прийняття рішень

* Необхідність швидкого запуску MVP.
* Мінімізація витрат на сторонні сервіси (Algolia / Meilisearch / Typesense).

## Розглянуті варіанти

1. Інтеграція Laravel Scout з драйвером `database`.
2. Реалізація власної логіки через кастомні `LIKE` запити.

## Результат рішення

Ми вирішили обрати **Варіант 1**.

### Наслідки

#### Плюси

- Можливість швидкої міграції на інший сервіс.
- Швидка реалізація функціоналу.

#### Мінуси

- Немає стійкості до друкарських помилок та синонімів.
- Навантаження йде на основу БД, а не на сторонній сервіс.
