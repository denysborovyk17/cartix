# ADR 0002: Кастомна система скидання пароля без готових пакетів

## Контекст і постановка проблеми
Потрібно реалізувати функціонал скидання пароля. Потрібно визначити, чи використовувати готові пакети екосистеми Laravel (Breeze / Fortify), чи написати логіку самостійно через кастомні контролери.

## Драйвери прийняття рішень
* Бажання глибоко розібратися в "підкапотному" механізмі роботи токенів Laravel.
* Уникнення зайвих файлів, які приносить з собою Breeze.

## Розглянуті варіанти
1. Інтеграція Breeze.
2. Реалізація власної логіки через кастомні контролери та вбудований сервіс `Password`.

## Результат рішення
Ми вирішили обрати **Варіант 2**. Оскільки нам не потрібен весь інший функціонал Breeze, кастомні контролери дозволяють тримати кодову базу чистою та контрольованою.

### Наслідки
* **Плюси:** Повний контроль над кожним кроком процесу, відсутність зайвого функціоналу який не використовується, максимальне розуміння системи автентифікації.
* **Мінуси:** Витрачено більше часу на ручне написання валідації, форм та контролерів.
