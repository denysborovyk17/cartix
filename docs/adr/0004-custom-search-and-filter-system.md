# ADR 0004: Пошукова система та фільтрація за допомогою Eloquent ORM

## Контекст і постановка проблеми

У процесі реалізації фільтрації товарів було виявлено, що рішення, закладене в - [ADR 0003](0003-laravel-scout-search-system.md), унеможливлює можливість побудови складних запитів (`whereHas`) через обмеженість функціоналу Laravel Scout.

При спробі об'єднати фільтри у межах Scout виникали баги (система некоректно видавала результати).

## Драйвери прийняття рішень

* Необхідність забезпечити коректну роботу фільтрів при будь-яких комбінаціях.
* Бажання позбутися невикористовуваного коду та залежностей якщо інструмент не виконує потрібну задачу.

## Розглянуті варіанти

1. Залишити `Laravel Scout`.
2. Видалення `Laravel Scout` та перехід на чистий `Eloquent ORM` із використанням DTO.

## Результат рішення

Ми вирішили обрати **Варіант 2**.

### Наслідки

#### Плюси

- Кодова база очищена від непотрібних трейтів та конфігів Scout. Логіка фільтрації інкапсульована в репозиторії `ProductRepository` з використанням `ProductSearchFilterData` (DTO).
- Повний доступ до можливостей `Eloquent ORM` для подальшого розширення фільтрів.

#### Мінуси

- Перехід на зовнішні системи (Meilisearch / Elastic) у майбутньому вимагатиме повного переписування репозиторію.
- Немає стійкості до друкарських помилок та синонімів через використання оператора `LIKE`, зниження продуктивності БД при масштабуванні каталогу.
