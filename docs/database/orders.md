# Домен: Orders Domain

## Опис домену

Даний домен відповідає за збереження інформації про замовлення користувачів.

## Основні бізнес-правила домену

* Перше головне правило > замовлення повинно містити хоча б один товар.
* Друге головне правило > сума замовлення дорівнює сумі всіх товарів з таблиці `order_items`.
* Третє головне правило > зберігається інформація про товар, що дозволяє в майбутньому бачити знімок товару замовлення на момент його створення.

## Архітектурний контекст

### Типові сценарії використання

1. Оформлення замовлення.
2. Перегляд вмісту замовлення.
3. Розрахунок вартості замовлення.

## Таблиця: orders

Зберігає інформацію про статус, суму замовлення та його особи.

### Основні поля

| Поле         | Тип               | NULL  | Foreign Key (FK) | Default   | Опис                                                 |
|:-------------|:------------------|:------|:-----------------|:----------|:-----------------------------------------------------|
| `id`         | `bigint unsigned` | `NO`  |                  |           | Унікальний ідентифікатор замовлення.                 |
| `user_id`    | `bigint unsigned` | `YES` | `users(id)`      |           | Унікальний ідентифікатор авторизованого користувача. |
| `status`     | `varchar(255)`    | `NO`  |                  | `pending` | Статус замовлення.                                   |
| `total`      | `int unsigned`    | `NO`  |                  |           | Сума замовлення.                                     |
| `first_name` | `varchar(255)`    | `NO`  |                  |           | Ім'я особи.                                          |
| `last_name`  | `varchar(255)`    | `NO`  |                  |           | Прізвище особи.                                      |
| `email`      | `varchar(255)`    | `NO`  |                  |           | Електрона адреса особи.                              |
| `phone`      | `varchar(255)`    | `NO`  |                  |           | Номер телефону особи.                                |
| `city`       | `varchar(255)`    | `NO`  |                  |           | Місто прибуття.                                      |
| `address`    | `varchar(255)`    | `NO`  |                  |           | Адреса прибуття.                                     |
| `notes`      | `text`            | `YES` |                  |           | Коментар до замовлення                               |
| `created_at` | `timestamp`       | `YES` |                  |           | Дата створення запису.                               |
| `updated_at` | `timestamp`       | `YES` |                  |           | Дата оновлення запису.                               |

### Зв'язки (Relationships)

* `Many To One` > `users` Багато замовлень може належати до одного користувача.
* `One To Many` > `order_items` Замовлення може мати багато предметів.

### Обмеження цілісності (Constraints)

* **Foreign Key: `user_id` REFERENCES `users(id)` ON DELETE SET NULL (якщо видалити користувача, поле `user_id` стає `null`, але замовлення не видаляється).**

### Особливості продуктивності

* *Можливі оптимізації:* Індексація поля `status` (ще не реалізовано).

### Де реалізовано

- `app/Models/Order/Order`.

## Таблиця: order_items

Зберігає інформацію про предмети замовлення.

### Основні поля

| Поле                 | Тип               | NULL  | Foreign Key (FK)       | Default | Опис                                          |
|:---------------------|:------------------|:------|:-----------------------|:--------|:----------------------------------------------|
| `id`                 | `bigint unsigned` | `NO`  |                        |         | Унікальний ідентифікатор предмету замовлення. |
| `order_id`           | `bigint unsigned` | `NO`  | `orders(id)`           |         | Унікальний ідентифікатор замовлення.          |
| `product_variant_id` | `bigint unsigned` | `NO`  | `product_variants(id)` |         | Унікальний ідентифікатор варіанта товару.     |
| `product_name`       | `varchar(255)`    | `NO`  |                        |         | Назва товару на момент створення замовлення.  |
| `quantity`           | `int`             | `NO`  |                        | `1`     | Кількість товару.                             |
| `price`              | `int unsigned`    | `NO`  |                        |         | Ціна товару на момент створення замовлення.   |
| `created_at`         | `timestamp`       | `YES` |                        |         | Дата створення запису.                        |
| `updated_at`         | `timestamp`       | `YES` |                        |         | Дата оновлення запису.                        |

### Зв'язки (Relationships)

* `Many To One` > `orders` Багато предметів замовлення можуть належати до одного замовлення.
* `Many To One` > `product_variants` Багато предметів замовлення можуть належати до одного варіанта товару.

### Обмеження цілісності (Constraints)

* **Foreign Key: `order_id` REFERENCES `orders(id)` ON DELETE CASCADE (якщо видалити замовлення, його предмети видаляються).**
* **Foreign Key: `product_variant_id` REFERENCED `product_variants(id)` ON DELETE RESTRICT (якщо спробувати видалити варіант товару, виникне помилка).**

### Де реалізовано

- `app/Models/Order/OrderItem`.
