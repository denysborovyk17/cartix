# Домен: Users Domain

## Опис домену

Даний домен відповідає за збереження користувачів та їх прив'язку до різних таблиць.

## Основні бізнес-правила домену

* Перше головне правило > `email` користувача має бути унікальним.
* Друге головне правило > пароль (`password`) користувача повинен бути хешованим.

## Архітектурний контекст

### Типові сценарії використання

1. Реєстрація та автентифікація
2. Авторизація за ролями

---

## Таблиця: users

Зберігає базову інформацію про зареєстрованих користувачів.

### Основні поля

| Поле                | Тип               | NULL  | Default | Опис                                  |
|:--------------------|:------------------|:------|:--------|:--------------------------------------|
| `id`                | `bigint unsigned` | `NO`  |         | Унікальний ідентифікатор користувача. |
| `name`              | `varchar(255)`    | `NO`  |         | Ім'я користувача.                     |
| `email`             | `varchar(255)`    | `NO`  |         | Електронна адреса користувача.        |
| `email_verified_at` | `timestamp`       | `YES` |         | Підтвердження електронної адреси.     |
| `password`          | `varchar(255)`    | `NO`  |         | Хешований пароль користувача.         |
| `remember_token`    | `varchar(100)`    | `YES` |         | Токен для системи Remember Me.        |
| `created_at`        | `timestamp`       | `YES` |         | Дата створення запису.                |
| `updated_at`        | `timestamp`       | `YES` |         | Дата оновлення запису.                |
| `role`              | `varchar(255)`    | `NO`  | `user`  | Роль користувача.                     |

### Зв'язки (Relationships)

* `One To One` > `carts` Користувач може мати лише один кошик.
* `One To Many` > `orders` Користувач може мати багато замовлень.
* `One To Many` > `wishlist_items` Користувач може мати багато товарів у списку бажань.
* `One To Many` > `reviews` Користувач може залишити лише один відгук про товар.

### Індекси (Indexes)

* `UNIQUE(email)` *Причина:* `email` повинен бути унікальним.

### Особливості продуктивності

* *Можливі оптимізації:* Індексація поля `role` (ще не реалізовано).

### Де реалізовано

- `app/Models/User/User`.

## Таблиця: wishlist_items

Зберігає інформацію про товари користувачів в списку бажань.

### Основні поля

| Поле                 | Тип               | NULL  | Foreign Key (FK)       | Опис                                             |
|:---------------------|:------------------|:------|:-----------------------|:-------------------------------------------------|
| `id`                 | `bigint unsigned` | `NO`  |                        | Унікальний ідентифікатор запису в списку бажань. |
| `user_id`            | `bigint unsigned` | `NO`  | `users(id)`            | Унікальний ідентифікатор користувача.            |
| `product_variant_id` | `bigint unsigned` | `NO`  | `product_variants(id)` | Посилання на варіант товару.                     |
| `created_at`         | `timestamp`       | `YES` |                        | Дата створення запису.                           |
| `updated_at`         | `timestamp`       | `YES` |                        | Дата оновлення запису.                           |

### Зв'язки (Relationships)

* `Many To One` > `users` Багато записів у списку бажань можуть належати одному користувачу.
* `Many To One` > `product_variants` Багато записів у списку бажань можуть посилатися на один і той самий варіант товару.

### Індекси (Indexes)

* `UNIQUE(user_id, product_variant_id)` *Причина:* забезпечення того, що один користувач не може мати 2 однакових товари в списку бажань.

### Обмеження цілісності

* **Foreign Key: `user_id` REFERENCES `users(id)` ON DELETE CASCADE (якщо видалити користувача, його список бажань зникне).**
* **Foreign Key: `product_variant_id` REFERENCES `product_variants(id)` ON DELETE CASCADE (якщо товар видаляють з магазину, він зникає з списку бажань).**
