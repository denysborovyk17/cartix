# Домен: Carts Domain

## Опис домену

Даний домен відповідає за збереження інформації в кошиках користувачів.

## Основні бізнес-правила домену

* Перше головне правило > авторизований користувач може мати лише один активний кошик.
* Друге головне правило > якщо користувач не авторизований, використовується ідентифікатор його сесії.

## Архітектурний контекст

### Типові сценарії використання

1. Додавання товару до кошика.
2. Зміна кількості товару.
3. Видалення товару з кошика. 
4. Розрахунок вартості товарів та кошика.

---

## Таблиця: carts

Зберігає базову інформацію про кошики користувачів.

### Основні поля

| Поле         | Тип               | NULL  | Foreign Key (FK) | Опис                                  |
|:-------------|:------------------|:------|:-----------------|:--------------------------------------|
| `id`         | `bigint unsigned` | `NO`  |                  | Унікальний ідентифікатор кошика.      |
| `user_id`    | `bigint unsigned` | `YES` | `users(id)`      | Унікальний ідентифікатор користувача. |
| `session_id` | `varchar(255)`    | `YES` |                  | Унікальний ідентифікатор сесії.       |
| `created_at` | `timestamp`       | `YES` |                  | Дата створення запису.                |
| `updated_at` | `timestamp`       | `YES` |                  | Дата оновлення запису.                |

### Зв'язки (Relationships)

* `One To One` > `users` Кошик може належати лише одному користувачу.
* `One To Many` > `cart_items` Кошик може мати багато предметів.

### Обмеження цілісності (Constraints)

* **Foreign Key: `user_id` REFERENCES `users(id)` ON DELETE CASCADE (якщо видалити користувача, його кошик видаляється).**

### Де реалізовано

- `app/Models/Cart/Cart`.

## Таблиця: cart_items

Зберігає інформацію про товари в кошиках користувачів.

### Основні поля

| Поле                 | Тип               | NULL  | Foreign Key (FK)       | Опис                                        |
|:---------------------|:------------------|:------|:-----------------------|:--------------------------------------------|
| `id`                 | `bigint unsigned` | `NO`  |                        | Унікальний ідентифікатор предмета в кошику. |
| `cart_id`            | `bigint unsigned` | `NO`  | `carts(id)`            | Унікальний ідентифікатор кошика.            |
| `product_variant_id` | `bigint unsigned` | `NO`  | `product_variants(id)` | Унікальний ідентифікатор варіанта товару.   |
| `quantity`           | `int unsigned`    | `NO`  |                        | Кількість варіанта товару.                  |
| `created_at`         | `timestamp`       | `YES` |                        | Дата створення запису.                      |
| `updated_at`         | `timestamp`       | `YES` |                        | Дата оновлення запису.                      |

### Зв'язки (Relationships)

* `Many To One` > `carts` Багато предметів в кошику належать до одного кошика.
* `Many To One` > `product_variants` Один предмет в кошику належить до одного варіанта товару.

### Обмеження цілісності (Constraints)

* * **Foreign Key: `cart_id` REFERENCES `carts(id)` ON DELETE CASCADE (якщо видалити кошик, його предмети видаляються).**
* * **Foreign Key: `product_variant_id` REFERENCES `product_variants(id)` ON DELETE CASCADE (якщо видалити варіант товару, його запис видаляється).**

### Де реалізовано

- `app/Models/Cart/CartItem`.
